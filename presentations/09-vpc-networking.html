What is a VPC?

Virtual Private Cloud — your own private network inside Google Cloud

Structure
VPC (your private network)
 ├── Subnet-1 (Mumbai — 10.0.1.0/24)
 ├── Subnet-2 (US — 10.0.2.0/24)
 └── Resources (VMs, DBs, Cloud Run)
Think of it as your company's office network, but in the cloud. You control who can enter. Outsiders can't see what's inside.
Without VPC

Everything public. No isolation. Anyone can reach your database. Security nightmare.

With VPC

Private communication. Controlled access. Database hidden from internet. Secure by design.

The Architecture That's Secure

Public entry, private backend — the golden rule

User
↓
Load Balancer (Public — the ONLY public entry point)
↓
App Server (Private IP — inside VPC)
↓
Database (Private IP — inside VPC)
Golden Rule: Only expose the Load Balancer. Everything else stays private.

Firewall Rules: Your Security Gates

Control what traffic can enter or leave your VPC

RuleProtocolPortPurpose
allow-sshTCP22SSH access to VMs
allow-httpTCP80Web traffic
allow-appTCP8080Application port
allow-healthTCPvariesLB health checks
Default: All incoming traffic is DENIED. You must explicitly allow what you need.
bash
gcloud compute firewall-rules \
  create allow-http \
  --network=my-vpc \
  --allow tcp:80 \
  --source-ranges=0.0.0.0/0

Hands-On: Build a VPC

Create VPC, subnet, firewall rule, and launch a VM

1. Create VPC
gcloud compute networks create \
  my-vpc --subnet-mode=custom
2. Create Subnet
gcloud compute networks subnets \
  create app-subnet \
  --network=my-vpc \
  --range=10.0.1.0/24 \
  --region=asia-south1
3. Add Firewall Rule
gcloud compute firewall-rules \
  create allow-ssh \
  --network=my-vpc \
  --allow tcp:22
4. Launch VM in VPC
gcloud compute instances create \
  app-vm \
  --network=my-vpc \
  --subnet=app-subnet \
  --zone=asia-south1-a

No firewall rulesOpen system, anyone connectsExplicit allow rules only
Wrong subnet planningIP conflicts, routing issuesPlan IP ranges upfront
Direct DB exposureData theft, ransomwareAlways use private IP for databases
Flat networkLateral movement in breachesSeparate subnets by function

Expert Takeaways

What you should remember from this section

  • VPC = the foundation of all cloud security
  • Subnets = IP segmentation by region and function
  • Firewalls = your control layer — deny by default, allow explicitly
  • Design pattern: Public entry → Private backend (always)
  • Networking = architecture backbone — get this right and everything else is easier
