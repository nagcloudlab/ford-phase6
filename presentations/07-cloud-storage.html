Storage Classes: The Cost Lever

Not all data is accessed equally — pay accordingly

ClassAccess PatternCostExample
StandardFrequently accessedHighestUser profile images
Nearline~Once a monthMediumMonthly reports
Coldline~Once a quarterLowCompliance archives
Archive~Once a yearLowest7-year audit logs
Cost savings example: Moving 1TB of old logs from Standard → Coldline can save ~60% on storage costs.

Access Control: Who Can See Your Files?

IAM-based security — private by default

IAM Roles (Recommended)

Storage Admin

Full control over buckets & objects

Object Viewer

Read-only access to files

Object Creator

Upload only — can't read or delete

Public vs Private

ModeAccess
Private (Default)Only authorized users. Use signed URLs for temp access.
PublicAnyone on internet via direct URL
Best Practice: Default everything to PRIVATE. Open up selectively.

Hands-On: Working with Buckets

Create, upload, list, and download — all from the command line

Create a Bucket
gcloud storage buckets create \
  gs://myapp-user-images \
  --location=asia-south1
Upload a File
gcloud storage cp photo.jpg \
  gs://myapp-user-images/
List Files
gcloud storage ls \
  gs://myapp-user-images/
Download a File
gcloud storage cp \
  gs://myapp-user-images/photo.jpg \
  ./local-photo.jpg

Upload user profile images — secure by design

Java

    .getDefaultInstance().getService();
BlobId blobId = BlobId.of(
    "myapp-bucket", "profile.png");

Standard for everythingOverpaying for cold dataUse appropriate storage classes
Storing files in databaseSlow queries, bloated DBUse Cloud Storage for files
No lifecycle rulesStorage costs grow foreverSet auto-transition rules
Not using signed URLsEither too open or too lockedUse signed URLs for temp access

Expert Takeaways

What you should remember from this section

  • Cloud Storage = default choice for any file storage
  • Buckets are global — plan naming carefully
  • Storage class = your cost optimization lever — use it wisely
  • IAM controls access — keep everything private by default
  • Always separate: Files → Storage, Data → Database
