Deploy Spring Boot to Cloud Run

Four steps — from code to production URL

1. Create Dockerfile
FROM openjdk:17-jdk-slim
COPY target/app.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "/app.jar"]
2. Build & Push Image
docker build -t gcr.io/PROJECT/myapp .
docker push gcr.io/PROJECT/myapp
3. Deploy
gcloud run deploy myapp \
  --image gcr.io/PROJECT/myapp \
  --region asia-south1
4. Access Your App

https://myapp-abc123.a.run.app

No servers. No config. Just a URL.

Auto-Scaling & Cold Start

How scaling works and the one gotcha to know

How Scaling Decisions Happen

1
Monitor

Cloud Run watches incoming request rate

2
Compare

Checks against concurrency setting (default: 80)

3
Scale Up

Spins up new instances when needed

4
Scale Down

Back to zero when traffic stops

The Cold Start Problem

What: First request has extra delay (~1–5s for Java/Spring Boot)

Solutions:

✅
Set min-instances=1 — keeps one instance warm
✅
Optimize startup time — lazy loading, smaller images
✅
GraalVM native images — sub-second boot times
